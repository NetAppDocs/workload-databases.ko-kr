= Workload Factory에서 잘 설계된 데이터베이스 구성을 구현합니다.
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
구성 분석 인사이트와 권장 사항을 활용하여 NetApp Workload Factory를 통해 Microsoft SQL Server 및 Oracle 데이터베이스 구성에 대한 모범 사례를 구현할 수 있습니다. 잘 설계된 아키텍처 상태를 쉽게 검토하고, 데이터베이스 구성 문제를 파악하고, 안정성, 보안, 효율성, 성능 및 비용 측면에서 최적화되지 않은 구성을 수정할 수 있습니다.

또한 환경에 적용되지 않는 구성 분석을 해제하여 불필요한 알림과 부정확한 결과를 방지할 수 있습니다.

link:optimize-overview.html["Workload Factory의 구성 분석과 잘 설계된 상태에 대해 알아보세요."]



== 이 작업에 대해

Workload Factory는 데이터베이스 구성을 매일 분석합니다. 일일 분석을 통해 우수한 아키텍처 상태, 인사이트 및 권장 사항을 제공합니다. 구성 문제를 자동으로 수정하여 모범 사례를 준수할 수 있습니다.

Workload Factory 콘솔 내의 데이터베이스 인벤토리 또는 Well-architected 대시보드에서 문제를 검토하고 수정할 수 있습니다.

선택적으로, 자격 증명을 저장하거나 리소스를 등록하지 않고도 데이터베이스 구성에 대한 모범 사례를 검토하기 위해 일회성 평가를 수행할 수 있습니다.

.분석 대상
Workload Factory는 다음 구성의 잘 설계된 상태를 분석합니다.

Microsoft SQL Server 인스턴스의 경우:

* 스토리지 사이징: 스토리지 계층, 파일 시스템 여유 공간, 로그 드라이브 크기 및 TempDB 드라이브 크기가 포함됩니다
* 스토리지 레이아웃: 사용자 데이터 파일 배치, 로그 파일 배치 및 TempDB 배치가 포함됩니다
* 스토리지 구성: 용량 관리, 씬 프로비저닝, 계층화 정책, 스냅샷, Microsoft Multipath I/O(MPIO) 상태 및 MPIO 시간 초과 설정이 포함됩니다.
* 컴퓨팅: 크기 조정, 운영 체제 패치, RSS(수신 측 크기 조정), TCP 오프로딩, MTU 정렬과 같은 네트워크 어댑터 설정이 포함됩니다.
* 애플리케이션: Microsoft SQL Server 라이센스, Microsoft SQL Server 패치 및 MAXDOP 설정 포함
* 복원력: 로컬 스냅샷, FSx for ONTAP 백업, 지역 간 복제(CRR), Microsoft SQL 고가용성이 포함됩니다.
* 복제본: Workload Factory 내부 또는 외부에서 생성되어 60일 이상 된 복제본(샌드박스)을 새로 고치고 삭제하는 옵션이 포함되어 있습니다.


Oracle 데이터베이스의 경우:

* 저장소 크기 조정: 스왑 공간 할당 및 파일 시스템 헤드룸 포함
* 스토리지 구성: 용량 관리, 씬 프로비저닝, 계층화 정책, 스냅샷, 스토리지 효율성, Automatic Storage Management(ASM) 사용 여부와 관계없이 NFS 또는 iSCSI를 사용하는 배포에 대한 운영 체제 구성(Microsoft Multipath I/O(MPIO) 상태 및 설정 포함), 그리고 다음 dNFS 설정: dNFS 활성화, dNFS 일관된 IP 확인, dNFS 구성 파일 및 dNFS `nosharecache`
* 저장소 레이아웃: 리두 로그 배치, 임시 테이블스페이스 배치, 데이터 파일 배치, 아카이브 로그 배치, 제어 파일 배치 및 바이너리 배치, ASM 디스크 그룹 LUN 수 포함
* 컴퓨팅: 운영 체제 패치 포함
* 애플리케이션: Oracle Critical Patch Updates 포함
* 복원력: 지역 간 복제(CRR) 및 애플리케이션 정합성 보장 스냅샷 포함


Oracle용 아키텍처 분석 기능은 구성이 다른 구성에 의존하는 경우 알려주고 연결된 구성을 최적화하기 위한 조치를 나열합니다.



== 구성을 일회성으로 평가하십시오

자격 증명을 저장하거나 리소스를 등록하지 않고도 Microsoft SQL Server 인스턴스 또는 Oracle 데이터베이스의 아키텍처 설계 상태를 일회성으로 평가할 수 있습니다.

스크립트를 실행하고 결과를 여러 번 업로드할 수 있습니다. Workload Factory는 업로드할 때마다 아키텍처 설계 상태와 인사이트를 업데이트합니다.

. 중 하나를 사용하여 link:https://docs.netapp.com/us-en/workload-setup-admin/console-experiences.html["콘솔 환경"^]로그인합니다.
. 메뉴 image:workloads-menu-icon.png["햄버거 메뉴 아이콘은 스토리지, EDA, AI, 데이터베이스, VMware 및 관리와 같은 워크로드로 이동하는 데 사용됩니다."]를 선택한 다음 *데이터베이스*를 선택합니다.
. *인벤토리*에서 엔진 유형으로 *SQL Server*를 선택하십시오.
. *일회성 평가* > *평가에 대해 알아보기*를 선택하세요. 일회성 평가에 대한 정보를 확인하세요.

. 모든 구성에 대한 권장 사항을 확인합니다. Workload Factory는 구성이 최적화되지 않은 경우 권장 사항에 모범 사례 및 잠재적 문제를 포함합니다.

== 구성을 수정, 해제 및 재활성화하기 전에

데이터베이스 환경을 지속적으로 모니터링하고 구성 문제를 자동으로 수정하려면 다음 요구 사항을 충족해야 합니다. 이러한 요구 사항은 일회성 평가에는 필요하지 않습니다.

* 당신은해야합니다link:https://docs.netapp.com/us-en/workload-setup-admin/add-credentials.html["_운영 및 수정_ 권한 부여"^] 귀하의 AWS 계정에서.
* 리소스는 Workload Factory에 등록되어 있어야 하며 스토리지 유형은 FSx for ONTAP이어야 합니다.link:register-instance.html["리소스 등록 방법을 알아보세요."]
* 설정이나 구성을 수정하기 전에 각 권장 사항을 검토하십시오. RSS 및 MAXDOP의 경우, 운영 환경을 변경하기 전에 권장 설정을 테스트하여 성능을 확인하십시오.




== 구성 문제 해결

FSx for ONTAP 스토리지에서 실행되는 SQL Server 또는 Oracle 환경의 구성 문제를 해결합니다.


NOTE: 구성을 수정하면 다운타임이나 서비스 중단이 발생할 수 있습니다. 구성을 수정하기 전에 권장 사항을 주의 깊게 검토하십시오.

.단계
. 중 하나를 사용하여 link:https://docs.netapp.com/us-en/workload-setup-admin/console-experiences.html["콘솔 환경"^]로그인합니다.
. 메뉴 image:workloads-menu-icon.png["햄버거 메뉴 아이콘은 스토리지, EDA, AI, 데이터베이스, VMware 및 관리와 같은 워크로드로 이동하는 데 사용됩니다."]를 선택한 다음 *데이터베이스*를 선택합니다.
. *인벤토리*에서 엔진 유형을 선택합니다: Microsoft SQL Server 또는 Oracle.
. 특정 구성에 맞게 수정할 리소스를 선택합니다.
+
** Microsoft SQL Server를 선택한 경우 *인스턴스* 탭을 선택합니다.
** Oracle을 선택한 경우 *데이터베이스* 탭을 선택합니다.


. 인스턴스의 구성 문제를 보려면 *보기 및 수정*을 선택하세요.
. 인스턴스의 Well-Architected 상태 페이지에서 분석 결과를 검토합니다.
+
구성 문제를 범주, 하위 범주, 상태, 심각도 및 태그별로 필터링할 수 있습니다.

+
*PDF 내보내기*를 선택하여 결과 보고서를 다운로드하십시오.

. 드롭다운 화살표를 선택하여 모든 구성에 대한 권장 사항을 확인합니다. 권장 사항에는 모범 사례, 최적화되지 않은 구성의 잠재적 함정, 중요 고려 사항 등이 포함됩니다. 권장 사항을 주의 깊게 검토하십시오.
. 옵션이 제공되는 경우 구성 문제를 *보고 수정*하도록 선택합니다.
+
*모두 선택*이 기본값이지만, 수정할 특정 리소스를 선택할 수 있습니다.

+
.. 클론 정리를 제외한 모든 구성의 경우 권장 사항 세부 정보를 검토하여 문제 해결의 영향을 확인하십시오. 일부 복구 작업으로 인해 다운타임이나 서비스 중단이 발생할 수 있습니다.
.. 클론 정리의 경우 새로 고치거나 삭제할 클론 데이터베이스(샌드박스)를 선택합니다.
+
*** 복제본을 새로 고치면 소스 데이터베이스와 동기화됩니다.  새로 고침은 Workload Factory에서 생성된 복제본에만 사용할 수 있습니다.
*** 복제본을 삭제하면 영구적으로 제거되고, 저장 공간이 확보되며, 비용이 절감됩니다.  Workload Factory 내부 및 외부에서 생성된 복제본을 삭제할 수 있습니다.




. 구성 문제를 해결하려면 * 계속 * 을 선택하십시오.


.결과
Workload Factory가 문제 해결을 시작합니다.  작업 상태를 보려면 *작업 모니터링* 탭을 선택하세요.



== 데이터베이스 구성 분석 해제

데이터베이스 환경에 적용되지 않는 특정 데이터베이스 구성에 대한 분석을 무시하여 불필요한 경고 및 부정확한 최적화 결과를 방지하십시오. 무시한 구성 분석은 언제든지 다시 활성화할 수 있습니다.

.단계
. 중 하나를 사용하여 link:https://docs.netapp.com/us-en/workload-setup-admin/console-experiences.html["콘솔 환경"^]로그인합니다.
. 메뉴 image:workloads-menu-icon.png["햄버거 메뉴 아이콘은 스토리지, EDA, AI, 데이터베이스, VMware 및 관리와 같은 워크로드로 이동하는 데 사용됩니다."]를 선택한 다음 *데이터베이스*를 선택합니다.
. 데이터베이스 메뉴에서 *인벤토리*를 선택합니다.
. 인벤토리에서 데이터베이스 엔진 유형을 선택합니다: *Microsoft SQL Server* 또는 *Oracle*.
. 연기할 구성이 있는 SQL Server 인스턴스나 Oracle 데이터베이스로 이동한 다음, 작업 메뉴를 선택하고 *Well-architected*를 선택합니다.
. Well-architected 분석 페이지에서 연기할 구성까지 아래로 스크롤하고 작업 메뉴를 선택한 다음 *해제*를 선택합니다.
. 해제 구성 대화 상자에서 *해제* 옵션을 선택한 다음 *해제*를 선택하여 해제를 확인합니다.


.결과
Workload Factory는 구성 분석을 중지합니다.

언제든지 분석을 다시 활성화할 수 있습니다. 해당 구성은 더 이상 전체 최적화 점수에 포함되지 않습니다.



== 해제된 구성 분석을 다시 활성화합니다

해제된 구성 분석을 언제든지 다시 활성화할 수 있습니다. 하나 이상의 구성을 선택하여 다시 활성화할 수 있습니다.

.단계
. 중 하나를 사용하여 link:https://docs.netapp.com/us-en/workload-setup-admin/console-experiences.html["콘솔 환경"^]로그인합니다.
. 메뉴 image:workloads-menu-icon.png["햄버거 메뉴 아이콘은 스토리지, EDA, AI, 데이터베이스, VMware 및 관리와 같은 워크로드로 이동하는 데 사용됩니다."]를 선택한 다음 *데이터베이스*를 선택합니다.
. 데이터베이스 메뉴에서 *인벤토리*를 선택합니다.
. 인벤토리에서 데이터베이스 엔진 유형을 선택합니다: *Microsoft SQL Server* 또는 *Oracle*.
. 연기할 구성이 있는 SQL Server 인스턴스나 Oracle 데이터베이스로 이동한 다음, 작업 메뉴를 선택하고 *Well-architected*를 선택합니다.
. Well-architected 분석 페이지에서 *해제된 구성*을 선택하면 해제된 구성만 표시됩니다.
. 연기되거나 취소된 구성에 대한 구성 분석을 다시 시작하려면 *다시 활성화*를 선택합니다.


.결과
Workload Factory는 구성 분석을 다시 활성화하고 지금부터 매일 실행합니다.
